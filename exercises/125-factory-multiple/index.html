Understanding Multiple Closures

Each Factory Call Creates a New Closure

When you call a factory function multiple times, each call creates a completely new closure with its own private variables. These closures are completely independent - they don't share state.

function createCounter(name, initialValue = 0) { let count = initialValue; // NEW variable for each call! return { getName() { return name; }, getValue() { return count; }, increment() { count++; } }; } // Each call creates a NEW closure const counter1 = createCounter('Counter 1', 0); const counter2 = createCounter('Counter 2', 0); const counter3 = createCounter('Counter 3', 0); // Each has its OWN count variable counter1.increment(); // counter1's count = 1 counter2.increment(); // counter2's count = 1 counter2.increment(); // counter2's count = 2 console.log(counter1.getValue()); // 1 console.log(counter2.getValue()); // 2 console.log(counter3.getValue()); // 0 (never incremented)
Key Insight:

Variables in closures are not shared between different instances. Each factory call creates fresh variables.

Why Multiple Instances Matter

Multiple instances are essential for real-world applications where you need many independent objects:

  • User Accounts: Each user has their own private data
  • Shopping Carts: Each customer has their own items list
  • Game Characters: Each player has their own state
  • Database Connections: Each connection is independent
  • Event Handlers: Each element has its own listeners

Pattern: Counter Registry

When you have many counters, it's useful to organize them in a registry that can manage them all:

function createCounterRegistry() { const counters = {}; // Private registry return { register(name, initialValue = 0) { counters[name] = createCounter(name, initialValue); }, get(name) { return counters[name]; }, increment(name) { if (name in counters) { counters[name].increment(); } }, getValue(name) { return counters[name]?.getValue(); }, listCounters() { return Object.keys(counters); } }; } const registry = createCounterRegistry(); registry.register('views', 0); registry.register('clicks', 0); registry.register('submissions', 0); registry.increment('views'); registry.increment('views'); registry.increment('clicks'); console.log(registry.listCounters()); // ['views', 'clicks', 'submissions'] console.log(registry.getValue('views')); // 2 console.log(registry.getValue('clicks')); // 1

Pattern: Counter Groups

Group related counters together and get statistics:

function createCounterGroup(groupName) { const counters = {}; // Private to this group return { addCounter(name, initialValue = 0) { counters[name] = createCounter(name, initialValue); }, getCounter(name) { return counters[name]; }, getTotal() { return Object.values(counters) .reduce((sum, counter) => sum + counter.getValue(), 0); }, getSummary() { const summary = {}; Object.keys(counters).forEach(name => { summary[name] = counters[name].getValue(); }); return summary; } }; } const stats = createCounterGroup('Website Stats'); stats.addCounter('visits', 100); stats.addCounter('users', 50); stats.addCounter('purchases', 25); console.log(stats.getTotal()); // 175 console.log(stats.getSummary()); // { visits: 100, users: 50, purchases: 25 }

Proof of Independence

Here's a visual proof that each counter has its own closure:

const counters = []; // Create 5 independent counters for (let i = 0; i < 5; i++) { counters.push(createCounter(`Counter ${i}`, 0)); } // Modify them all independently counters[0].increment(); // 1 counters[1].increment(); // 1 counters[1].increment(); // 2 counters[3].increment(); // 1 counters[3].increment(); // 2 counters[3].increment(); // 3 // Each kept its own count! console.log(counters[0].getValue()); // 1 - independent console.log(counters[1].getValue()); // 2 - independent console.log(counters[2].getValue()); // 0 - never modified console.log(counters[3].getValue()); // 3 - independent console.log(counters[4].getValue()); // 0 - never modified // This is IMPOSSIBLE without closures!

Real-World Example: User Profiles

function createUserProfile(username) { const profile = { followers: 0, following: 0, posts: 0 }; return { follow() { profile.following++; }, unfollow() { if (profile.following > 0) profile.following--; }, post() { profile.posts++; }, getStats() { return { ...profile }; } }; } const alice = createUserProfile('Alice'); const bob = createUserProfile('Bob'); alice.follow(); alice.follow(); alice.post(); alice.post(); bob.follow(); bob.post(); console.log(alice.getStats()); // { following: 2, posts: 2 } console.log(bob.getStats()); // { following: 1, posts: 1 } // Alice's and Bob's data are completely separate! // No way they can accidentally interfere with each other
Pro Tip:

When you have lots of instances, consider using a registry or manager object to keep track of them. It makes it easier to query and update groups of instances.

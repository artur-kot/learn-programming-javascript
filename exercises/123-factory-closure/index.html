Understanding Closures and Factory Functions

What is a Closure?

A closure is a function that "remembers" the variables from the scope where it was created, even after that scope has finished executing. It's like a function carrying its own environment with it.

Key Concept:

Functions have access to variables in their parent scope, and they keep that access even after the parent function returns.

Simple Closure Example

function outer() { // This variable lives in outer's scope let count = 0; // inner() is a closure - it has access to count function inner() { count++; return count; } // Return the inner function return inner; } // Create a closure const counter = outer(); console.log(counter()); // 1 console.log(counter()); // 2 console.log(counter()); // 3 // count is NOT accessible here // console.log(count); // ‚ùå ReferenceError // But each call to counter() still accesses the same count

What is a Factory Function?

A factory function is a function that creates and returns objects. Instead of using the new keyword with constructors, factories create objects in a more flexible way. They're "factories" because they manufacture objects on demand.

Why Use Factories?
  • More flexible than constructors
  • Can easily create private variables
  • Easier to test
  • More intuitive syntax
Factory Function Example
function createCounter(initialValue = 0) { // This is a private variable // Only accessible inside this function and its closures let count = initialValue; // Return an object with public methods return { // These methods form closures around the count variable increment() { count++; return this; }, decrement() { count--; return this; }, getValue() { return count; } }; } // Create two independent counters const counter1 = createCounter(0); const counter2 = createCounter(100); counter1.increment(); counter1.increment(); console.log(counter1.getValue()); // 2 counter2.decrement(); console.log(counter2.getValue()); // 99 // Each counter has its own private count!

Closures and State Encapsulation

The real power of closures is encapsulation - hiding private data that can only be accessed through public methods. This creates a safe, controlled interface.

function createBankAccount(initialBalance) { // Private variable - can ONLY be accessed through methods let balance = initialBalance; // Private function - only used internally function logTransaction(type, amount) { console.log(`${type}: $${amount}`); } // Public interface return { deposit(amount) { if (amount > 0) { balance += amount; logTransaction('Deposit', amount); } }, withdraw(amount) { if (amount > 0 && amount <= balance) { balance -= amount; logTransaction('Withdraw', amount); } }, getBalance() { return balance; } }; } const myAccount = createBankAccount(1000); myAccount.deposit(500); // Deposit: $500 myAccount.withdraw(200); // Withdraw: $200 console.log(myAccount.getBalance()); // 1300 // You CANNOT do: // myAccount.balance = 999999; // ‚ùå balance is private! // myAccount.logTransaction('Fraud', 10000); // ‚ùå private function! // This is the power of closures: data protection

Different Types of Factories

1. Basic Factory
function createCounter() { let count = 0; return { increment: () => count++, getValue: () => count }; } const counter = createCounter(); counter.increment(); console.log(counter.getValue()); // 1
2. Factory with Parameters
function createCounter(start = 0, step = 1) { let count = start; return { increment: () => count += step, decrement: () => count -= step, getValue: () => count }; } const fast = createCounter(0, 10); const slow = createCounter(0, 1); fast.increment(); // count = 10 slow.increment(); // count = 1
3. Factory with Configuration
function createBoundedCounter(config) { let count = config.initial; return { increment() { if (count < config.max) count++; return count; }, decrement() { if (count > config.min) count--; return count; }, getValue() { return count; } }; } const bounded = createBoundedCounter({ initial: 5, min: 0, max: 10 }); bounded.increment(); // 6 bounded.increment(); // 7
4. Factory with Private Methods
function createLogger() { let logs = []; // Private function function formatLog(message) { return `[${new Date().toISOString()}] ${message}`; } // Public interface return { log(message) { logs.push(formatLog(message)); }, getLogs() { return [...logs]; // Return copy to prevent external modification }, clear() { logs = []; } }; } const logger = createLogger(); logger.log('User logged in'); logger.log('Page loaded'); console.log(logger.getLogs());

Why Closures Matter

  • Data Privacy: Variables are truly private - impossible to access directly
  • State Management: Each instance maintains its own state independently
  • Encapsulation: Hide implementation details behind public methods
  • Flexibility: Create objects without class syntax
  • Functional Programming: Functions as first-class objects with behavior
üí° Pro Tip:

Closures are everywhere in JavaScript! Array methods like forEach, map, and filter all use closures to maintain state and context.

Common Closure Mistakes

// ‚ùå WRONG: Not using factory pattern let count = 0; const increment = () => count++; const decrement = () => count--; // Problem: count is global and can be modified anywhere! count = 999; // Oops! // ‚úÖ RIGHT: Using factory with closures function createCounter() { let count = 0; // Private! return { increment: () => count++, decrement: () => count--, getValue: () => count }; } const counter = createCounter(); // count cannot be accessed or modified directly

Real-World Factory Examples

Factories are used everywhere in professional code:

Database Connection Factory
function createDatabase(config) { let isConnected = false; return { connect() { isConnected = true; console.log(`Connected to ${config.host}`); }, query(sql) { if (!isConnected) throw new Error('Not connected'); return `Results for: ${sql}`; }, disconnect() { isConnected = false; } }; }
API Request Factory
function createApiClient(baseURL) { let requestCount = 0; return { async get(endpoint) { requestCount++; const response = await fetch(baseURL + endpoint); return response.json(); }, getRequestCount() { return requestCount; } }; } const api = createApiClient('https://api.example.com'); await api.get('/users'); // requestCount = 1 await api.get('/posts'); // requestCount = 2

Advanced Counter Patterns

1. Counter with Getters and Setters

Use JavaScript getter/setter syntax to create a clean API for accessing and modifying counter values with validation built in.

function createAdvancedCounter(name, initialValue, max, min) { let count = initialValue; const initial = initialValue; return { // Getter - read-only syntax get value() { return count; }, // Setter - write-only syntax set value(newValue) { if (newValue > max) count = max; else if (newValue < min) count = min; else count = newValue; }, increment() { this.value++; // Uses setter! }, reset() { count = initial; } }; } const counter = createAdvancedCounter('Test', 5, 10, 0); console.log(counter.value); // 5 counter.value = 12; // Respects max console.log(counter.value); // 10

2. Observable Counter with Callbacks

Create a counter that notifies observers whenever the value changes using closures to maintain an array of callbacks.

function createObservableCounter(initialValue) { let count = initialValue; const observers = []; // Store callbacks function notifyObservers(oldValue, newValue) { observers.forEach(observer => observer(oldValue, newValue)); } return { getValue() { return count; }, increment() { const oldValue = count; count++; notifyObservers(oldValue, count); }, // Subscribe to changes onChange(callback) { observers.push(callback); // Return unsubscribe function return () => { observers.splice(observers.indexOf(callback), 1); }; } }; } const counter = createObservableCounter(0); const unsubscribe = counter.onChange((old, newVal) => { console.log(`Changed from ${old} to ${newVal}`); }); counter.increment(); // "Changed from 0 to 1" counter.increment(); // "Changed from 1 to 2" unsubscribe(); // Stop listening counter.increment(); // No log

3. Counter with History and Undo/Redo

Track all changes to a counter and allow undoing and redoing operations using separate stacks for history management.

function createHistoryCounter(initialValue) { let count = initialValue; const undoStack = []; const redoStack = []; return { getValue() { return count; }, increment() { undoStack.push(count); count++; redoStack.length = 0; // Clear redo when new operation }, decrement() { undoStack.push(count); count--; redoStack.length = 0; }, undo() { if (undoStack.length === 0) return; redoStack.push(count); count = undoStack.pop(); }, redo() { if (redoStack.length === 0) return; undoStack.push(count); count = redoStack.pop(); }, canUndo() { return undoStack.length > 0; }, canRedo() { return redoStack.length > 0; }, getHistory() { return [...undoStack]; // Copy of history } }; } const counter = createHistoryCounter(0); counter.increment(); // 1 counter.increment(); // 2 counter.increment(); // 3 counter.undo(); // Back to 2 counter.undo(); // Back to 1 counter.redo(); // Forward to 2

4. Validated Counter

Use closures to enforce custom validation rules when changing the counter value.

function createValidatedCounter(initialValue, validator) { let count = initialValue; // Default validator: no negative numbers const defaultValidator = (value) => value >= 0; const validate = validator || defaultValidator; return { getValue() { return count; }, increment() { const newValue = count + 1; if (validate(newValue)) { count = newValue; } }, decrement() { const newValue = count - 1; if (validate(newValue)) { count = newValue; } }, setValue(newValue) { if (validate(newValue)) { count = newValue; } } }; } // Even numbers only const evenValidator = (value) => value % 2 === 0; const counter = createValidatedCounter(0, evenValidator); counter.increment(); // 0 (1 is odd, rejected) counter.increment(); // 0 (1 is odd, rejected) counter.increment(); // 2 (2 is even, accepted)

5. Combining Advanced Features

You can combine multiple advanced patterns into a single counter:

function createPremiumCounter(initialValue, max, min) { let count = initialValue; const history = []; const observers = []; const initial = initialValue; function notify(oldVal, newVal) { observers.forEach(obs => obs(oldVal, newVal)); } return { getValue() { return count; }, increment() { const oldVal = count; if (count < max) { count++; history.push(oldVal); notify(oldVal, count); } }, reset() { const oldVal = count; count = initial; history.length = 0; notify(oldVal, count); }, onChange(callback) { observers.push(callback); return () => { observers.splice(observers.indexOf(callback), 1); }; }, getHistory() { return [...history]; } }; } const premium = createPremiumCounter(0, 20, 0); premium.onChange((old, newVal) => { console.log(`${old} â†’ ${newVal}`); }); premium.increment(); premium.increment(); premium.reset();
Pro Pattern:

Real-World Use Cases

  • Undo/Redo: Text editors, drawing apps - use history pattern
  • Form Validation: Ensure values meet constraints - use validated pattern
  • State Changes: Notify UI when data changes - use observable pattern
  • Audit Trail: Track all changes for logging - use history pattern
  • Analytics: Event tracking with callbacks - use observable pattern

Key Principles

  • Encapsulation: Hide implementation details behind clean API
  • Separation of Concerns: Each pattern handles one responsibility
  • Composability: Patterns can be combined and extended
  • Testability: Private closures make testing behavior easy
